# Summarize

Condense long documents, PDFs, meeting notes, articles, or any lengthy content into clear, actionable summaries.

## Activation

Use when: user asks to "summarize", "condense", "give me the key points",
"TLDR", "what does this say", or shares a long document for review.

## Workflow

1. **Read**: Ingest the full document or content
2. **Extract**: Identify key points, decisions, action items, and critical data
3. **Structure**: Organize into a clear summary format
4. **Deliver**: Save as document or present inline (based on length)

## Summary Formats

Choose based on content type:
- **Meeting notes** → Action items, decisions, owners, deadlines
- **Report/article** → Executive summary (3-5 bullet points) + key findings
- **Long email thread** → What was decided, what's pending, who needs to act
- **Technical document** → Plain-language overview + critical details only
- **PDF/contract** → Key terms, obligations, dates, amounts

## Output Rules

- Keep summaries to 1 page max unless asked for more
- Use bullet points, not paragraphs
- Highlight action items and deadlines prominently
- If the original is a file, produce a summary file alongside it
- Always offer: "Want the full version with my annotations?"

## After Delivery

- "Want me to turn this into an email to the team?"
- "Should I extract just the action items as a task list?"
- "Want me to create a brief for leadership?"
