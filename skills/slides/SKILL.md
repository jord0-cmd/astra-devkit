# Slides

Build presentations with speaker notes, visual structure, and clear narrative flow.

## Activation

Use when: user asks for "slides", "presentation", "PowerPoint", "deck", "ppt",
or wants to present information visually.

## Workflow

1. **Clarify**: Topic, audience, number of slides, any brand/style requirements
2. **Outline**: Propose slide titles and key points before building
3. **Build**: Create using PowerPoint MCP — real .pptx file
4. **Deliver**: Save to workspace, show slide summary

## Slide Structure

Every presentation should include:
- Title slide (topic, presenter name if known, date)
- Agenda/overview slide
- Content slides (one key idea per slide, minimal text)
- Summary/takeaway slide
- Speaker notes on every slide (what to say, not what's on screen)

## Output Rules

- Use PowerPoint MCP → produce .pptx file
- Keep text minimal — bullet points, not paragraphs
- Suggest visuals: "I can add a chart here if you have the data"
- If PowerPoint MCP unavailable, create a structured Markdown outline

## After Delivery

- "Want me to add charts from your data?"
- "Should I create a handout version (Word doc with the same content)?"
- "Want me to research more on any of these topics?"
