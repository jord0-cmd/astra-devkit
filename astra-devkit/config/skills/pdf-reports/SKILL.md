# PDF Reports

Generate professional PDF reports from project data, analysis results, or structured content.

## Activation

Use when: user asks to "generate a PDF", "create a report", "make a PDF", "export to PDF",
or needs formatted document output beyond plain markdown.

## Strategy

PDFs are generated via a two-stage pipeline:
1. **Write HTML** with print-optimised CSS (stage you control)
2. **Convert to PDF** via the pandoc MCP server (automatic)

Do NOT use LaTeX. Do NOT use wkhtmltopdf. The pandoc MCP handles conversion.

## Pipeline

### Stage 1: Write the HTML Report

Create `report.html` with embedded CSS. The HTML MUST include:

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <style>
    /* Print-optimised CSS — see references/report-template.html */
  </style>
</head>
<body>
  <!-- Report content -->
</body>
</html>
```

### Stage 2: Convert via Pandoc MCP

Call the pandoc MCP `convert` tool:
- Input: the HTML file path
- Output format: pdf
- Output: desired PDF file path

If pandoc MCP is unavailable, fall back to shell: `pandoc report.html -o report.pdf --pdf-engine=weasyprint`

## Report Structure

Every report MUST include:
1. **Title page** — report name, date, author/org, version
2. **Table of contents** — auto-generated from h2/h3 headings
3. **Executive summary** — 2-3 paragraphs max
4. **Body sections** — structured with h2/h3 hierarchy
5. **Footer** — page numbers, generation date, "Generated by Astra DevKit"

## Styling Rules

1. Use `@media print` CSS for page breaks, margins, headers/footers
2. Page size: A4 (210mm x 297mm) with 20mm margins
3. Body font: system sans-serif, 11pt
4. Headings: bold, with bottom border for h2
5. Tables: alternating row colours, 1px solid borders
6. Code blocks: monospace, light grey background, 9pt
7. Charts/images: max-width 100%, centred, with captions
8. Page breaks: before each h2 section (`page-break-before: always`)

## Data Visualisation

For charts in reports:
- Generate SVG inline (not external files)
- Simple bar/line charts: hand-write SVG elements
- Complex charts: use a small inline script with Chart.js CDN (print-safe)
- Always include a text summary alongside visual data

## Rules

1. Never generate a PDF without the HTML intermediate — the user may want to edit it
2. Always show the user the HTML path before converting
3. Include generation metadata in the PDF (date, tool version, source)
4. Tables must not overflow page width — use word-wrap and smaller font if needed
5. Images must have alt text
6. The pandoc MCP is the primary conversion path. Shell pandoc is fallback only.
